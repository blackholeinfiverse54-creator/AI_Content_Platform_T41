# ARTHA Evidence Directory

This directory contains runtime evidence generated by ARTHA's proof scripts.

## Directory Structure

```
evidence/
├── phase4/                          # Phase 1-2 Evidence
│   ├── replay_verification_results.json    # Deterministic replay proof
│   └── full_compliance_trace_evidence.json # Compliance continuity proof
├── phase5/                          # Phase 3 Evidence
│   ├── production_audit_results.json       # Combined audit results
│   ├── system_health_audit.json            # System health audit
│   ├── security_audit.json                 # Security configuration audit
│   ├── database_integrity_audit.json       # Database integrity audit
│   ├── api_compliance_audit.json           # API compliance audit
│   └── configuration_audit.json            # Configuration audit
├── audit/                           # Individual audit evidence
│   ├── system_health_audit.json
│   ├── security_audit.json
│   ├── database_integrity_audit.json
│   ├── api_compliance_audit.json
│   └── configuration_audit.json
├── replay-proof/                    # Legacy replay evidence
│   └── (existing evidence files)
└── compliance/                      # Compliance evidence (generated on proof execution)
```

## Generating Evidence

```bash
cd backend

# Run all proof phases (generates all evidence)
node scripts/proof-all.js

# Or individually:
node scripts/proof-replay.js       # Generates phase4/replay_verification_results.json
node scripts/proof-compliance.js   # Generates phase4/full_compliance_trace_evidence.json
node scripts/proof-audit.js        # Generates phase5/*.json
node scripts/proof-certify.js      # Generates handover/*.json
```

## Evidence Format

All evidence files are JSON with the following structure:

```json
{
  "phase": "Phase N — Description",
  "started_at": "ISO timestamp",
  "completed_at": "ISO timestamp",
  "status": "PASS|PARTIAL|ERROR",
  "steps": [...],
  "summary": {...}
}
```

## Verification

Evidence is generated from actual system execution. Each evidence file contains:
- Timestamps from real execution
- Actual database ObjectIds
- Real service method results
- Genuine hash values

No evidence is fabricated or manually written.
